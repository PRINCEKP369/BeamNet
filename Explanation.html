BeamNet — Interactive Presentation Guide

Layer-by-layer mathematics · Q&A preparation · Presenter guide

Input
Normalized MVDR spectrum
Shape: (B, 1, 180)
The raw MVDR power at each of 180 degree bins is collected into a vector, then standardized before entering the network.
Normalization
x_norm = (x − μ) / (σ + ε)
μ = dataset mean (estimated on 10k subsample)
σ = dataset std,  ε = 1e-8 for numerical safety

Why? MVDR values are in arbitrary dB-like units.
Standardizing centers gradients near zero → faster convergence.
Block 1
MultiScaleFusion — entry block
3 parallel convolutions → concat → BN → GELU
Three 1D convolutions with different kernel sizes run in parallel, each capturing a different angular scale. Their outputs are concatenated into a 64-channel feature tensor.
Three parallel branches
F₃  = Conv1d(x; W₃,  k=3,  p=1)  ∈ ℝB×21×180
F₇  = Conv1d(x; W₇,  k=7,  p=3)  ∈ ℝB×21×180
F₁₅ = Conv1d(x; W₁₅, k=15, p=7) ∈ ℝB×22×180

z = Concat[F₃; F₇; F₁₅] ∈ ℝB×64×180
z = BatchNorm1d(z)
z = GELU(z)
BatchNorm1d
ẑ = (z − μB) / √(σ²B + ε)
out = γ·ẑ + β    (γ, β are learnable per-channel scalars)

Computed over the batch dimension. Stabilizes training by
keeping activations unit-scaled between blocks.
GELU activation
φ(z) = z · Φ(z)    where Φ = Gaussian CDF
≈ z · σ(1.702z)    (fast approximation used in practice)

Unlike ReLU, GELU is smooth and non-zero for all z.
Non-zero gradient everywhere → better flow in deep nets.
Blocks 2–7
DilatedResBlock ×6
Dilations d = 1, 2, 4, 8, 16, 1
Six residual blocks with increasing dilation. Each block has two dilated convolutions in its branch; the original input is added back (residual) before the final activation.
Dilated convolution — core formula
y[n] = Σk=0K−1 x[n + d·k] · w[k]

d = dilation factor (how far apart the taps are)
K = kernel size (= 3 throughout)

Effective receptive field per single layer:
r = 1 + 2·d·(K−1)  →  d=1:5°  d=2:9°  d=4:17°  d=8:33°  d=16:65°
Full block forward pass
h₁  = GELU( BN( DilConvd(x, W₁) ) )
h₂  = BN( DilConvd(h₁, W₂) )
out = GELU( x + h₂ )    ← residual add then activate

Residual gradient: ∂L/∂x = ∂L/∂out · (1 + ∂h₂/∂x)
The "+1" prevents vanishing gradients even at depth 6.
Cumulative receptive field (6 stacked blocks)
5 + 9 + 17 + 33 + 65 + 5 ≈ 134° effective angular span

A sidelobe 50° away from the mainlobe falls inside the RF.
The old model (RF≈9°) could not see it at all.
Blocks 8, 10, 12
ChannelAttention (SE gate) ×3
Applied after encoder pairs 1–2, 3–4, 5–6
Squeeze-and-Excitation attention: collapses all 180 spatial positions into a single importance score per channel, then rescales each channel by its score.
Step 1 — Squeeze (global average pool)
sc = (1/L) · Σl=1L z[c, l]    L=180
s ∈ ℝ64    (one scalar per channel)
Step 2 — Excitation (bottleneck MLP)
e = σ( W₂ · GELU( W₁ · s ) )
W₁ ∈ ℝ16×64,  W₂ ∈ ℝ64×16   (reduction r=4)
e ∈ [0,1]64    (importance score per channel)
Step 3 — Scale (channel-wise broadcast)
z̃[c, :] = ec · z[c, :]

Full expression:
z̃ = z ⊙ σ( W₂ · GELU( W₁ · GAP(z) ) ).unsqueeze(−1)

Channels responding to noise get e≈0 → suppressed.
Channels detecting peaks get e≈1 → amplified.
Output
Output head + Sigmoid
Shape: (B, 1, 180) → predictions ∈ (0, 1)
Head layers
z₁ = GELU( Conv1d(z, 64→32, k=3, p=1) )
z₂ = Conv1d(z₁, 32→1, k=1)    (pointwise projection)
ŷ  = σ(z₂) = 1 / (1 + e−z₂)  ∈ (0,1)180
BeamLoss = α·MSE + (1−α)·FocalBCE
MSE       = (1/N) Σ (ŷ − y)²

FocalBCE:
  wi  = pos_weight  if yi=1,  else 1
  pti = ŷi         if yi=1,  else (1−ŷi)
  Li  = −wi·(1−pti)γ·log(pti)

α=0.5,  γ=2.0,  pos_weight=100

pos_weight=100 ≈ (179 zero bins) / (1 peak bin).
Focal γ=2 down-weights easy negatives and focuses training
on the hard positives near decision boundaries.

Architecture
architecture
Why dilated convolutions instead of pooling + upsampling (U-Net style)? ▾
Pooling discards spatial position — critical for 1° angular resolution. If you pool and upsample, your output can be off by several bins. Dilation expands the receptive field without removing any positional information. Since knowing the exact angle bin is the entire point of the task, L=180 must be maintained throughout. U-Net is the right choice when you need to localize features at multiple resolutions (e.g. segmentation). Here, there is only one resolution: degree bins.
architecture
Why not a Transformer or LSTM for the 180-element angular sequence? ▾
The sequence is short (L=180). Transformers have O(L²) attention cost and require positional encoding because they have no inductive bias for locality. LSTMs are sequential — slow on GPU and they struggle with long-range structure unless carefully tuned. Dilated CNNs achieve comparable or larger receptive fields at O(L) cost, are fully parallelizable, and have a built-in local-to-global hierarchy that matches the angular structure of beamformer spectra. For L=180, the CNN wins on every axis: speed, memory, and inductive bias.
architecture
Why BatchNorm1d specifically — not LayerNorm or GroupNorm? ▾
BatchNorm computes statistics over the batch dimension, which works well with large fixed-length inputs (batch=512, length=180 — statistics are stable). LayerNorm normalizes per sample, which is better for variable-length sequences where batch statistics are noisy (e.g. NLP). GroupNorm is designed for small batches (e.g. in object detection) where BatchNorm statistics are unreliable. Neither of those conditions applies here, so BatchNorm is the correct and fastest choice.
architecture
What does channel attention actually learn to do? ▾
The SE gate learns to assign high importance scores (e≈1) to feature channels that activate strongly on sharp mainlobe peak patterns, and low scores (e≈0) to channels that respond to noise and diffuse sidelobe structure. This is feature selection without any explicit supervision — the gradient flows back through the gate weights and teaches it which channels are useful for predicting the sparse target. Applied 3× at progressively larger receptive field scales, each gate refines the selection at a different angular scale.
Training & Loss
training
Won't MSE/L1 work fine with 500k training samples? ▾
No — and more data makes this worse, not better. With ~1 positive per 180 bins, the gradient from the 179 zero-target bins dominates every update. The global loss minimum under MSE is simply "predict zero everywhere" (loss ≈ 1/180 ≈ 0.006). The network converges to this trivial solution quickly and stays there. More data just reinforces it. You need a loss that makes getting the peaks wrong disproportionately expensive — that is exactly what pos_weight=100 achieves in the focal BCE term.
training
How do you choose pos_weight=100? What if you have multiple simultaneous targets? ▾
Rule of thumb: pos_weight ≈ (number of negative bins) / (number of positive bins). For 1 target in 180 bins: 179/1 ≈ 100. For 3 simultaneous targets: 177/3 ≈ 59. In practice, monitor validation precision and recall — if recall is low (missing peaks), increase pos_weight; if precision is low (too many false peaks), decrease it. The focal exponent γ=2 additionally down-weights easy negatives (background confidently predicted at ≈0) so training focuses on the hard positives near decision boundaries.
training
Why AdamW over Adam? Why CosineAnnealing? ▾
Adam's L2 regularization is not true weight decay — the gradient scaling by the adaptive second moment means large-gradient parameters receive proportionally less regularization than intended. AdamW decouples weight decay from gradient scaling, making it consistent across all parameters. CosineAnnealing avoids sudden LR drops (which can cause loss spikes) and smoothly decays from 1e-3 to 1e-5 over 50 epochs. With 500k samples at batch=512, each epoch sees ~976 gradient steps — the slow decay allows the optimizer to settle into a flat minimum.
Challenging / Critical
challenging
Can this resolve two targets that are only 1–2° apart? ▾
The resolution ceiling is the bin resolution of the input (1°/bin). If the training data includes cases where two adjacent bins are both 1, the network can learn to output two separate peaks — the k=3 branch in the entry block is designed for exactly this fine-scale structure. However, if the MVDR input itself cannot separate two sources that close (its resolution depends on array aperture and SNR), then BeamNet cannot recover information that was already lost in the input. The limiting factor is MVDR resolution, not the neural network.
challenging
What are the known failure modes of BeamNet? ▾
Three main failure modes: (1) False peaks at low SNR — strong sidelobes can fool the channel attention into amplifying them, producing spurious predictions. (2) Edge-angle degradation — targets at 1° or 180° have asymmetric context (dilation taps fall outside the array), so peak sharpness may be reduced near boundaries; zero-padding partially compensates. (3) Over-smoothing — if α is set too high (too much MSE weight), output peaks become blurred rather than sharp. Monitor precision and recall at a 0.5 threshold on validation data to catch all three.
challenging
How does the model generalize to SNR values outside the training range? ▾
Two mechanisms help. First, input normalization standardizes absolute power levels — the network sees the shape of the spectrum, not its scale. Second, BatchNorm statistics are computed per-batch, providing additional implicit normalization that makes the model less sensitive to absolute amplitude. However, if the test SNR is far outside the training distribution, sidelobe patterns may look qualitatively different and the model may fail. The recommended mitigation is to include the full expected SNR range in training, and optionally apply test-time normalization per sample instead of global training statistics.
challenging
Why not a diffusion model or generative approach for this task? ▾
Diffusion models are stochastic at inference — they introduce random noise and then denoise. For angle estimation, you want a deterministic output: the same input should always map to the same predicted angle. Stochasticity adds uncertainty without benefit. Additionally, diffusion requires many forward passes (often 10–1000 denoising steps) at inference time, making it orders of magnitude slower than a single BeamNet forward pass. BeamNet gives you a deterministic prediction in one O(L) pass — ideal for real-time or near-real-time beamforming applications.

How to handle each part of your presentation — what to say, what to draw, what to expect.

Opening — set the problem first
Don't open with architecture. Open with the tension: "MVDR gives us a power value for every angle from 1° to 180°. The problem is sidelobes — false peaks that look like real sources. We trained a neural network to clean this up." Draw a rough 1×180 bar chart on the board: tall bar at the true angle, smaller bars scattered elsewhere. Ask: "How do you know which peak is real?" That question motivates everything.
Explaining the architecture pipeline
Walk left to right: Input → MultiScaleFusion → DilatedResBlocks ×6 → ChannelAttention ×3 → Output. For each block, lead with one sentence of intuition before showing math. For dilation, draw expanding circles on the board: "d=1 sees 5°, d=2 sees 9°, d=4 sees 17°…" — make the receptive field growth visible, not just numerical.
Presenting the math — what to say vs. what to write
Never read equations out loud. For each formula, name the variables first. For dilated conv: "x[n + d·k] means we skip d bins between each kernel tap — so at d=8, our 3-tap kernel spans 16 bins." For attention: "s_c is the average activation of channel c across all 180 angles — we're asking: is this channel globally active?" Write the formula on the board after the explanation, not before.
The loss slide — make the sparsity problem visceral
Ask the audience: "If your target is [0, 0, 0, 1, 0, …, 0], what does a network learn if you use mean squared error?" Pause. Then reveal: it predicts all zeros and gets MSE = 1/180 ≈ 0.006 — not bad by MSE standards, but completely useless. That's why we need pos_weight=100. The focal term is the follow-up: "We also want to focus training on the hard cases — the background bins the model is already getting right don't need more gradient."
The comparison table — let it do the work
Don't narrate every row. Say: "Every row in this table is a deliberate design choice, not a default." Then walk through the two most impactful ones: receptive field (9° vs 130°) and loss function (L1 vs focal BCE). For the others, say "I'm happy to go deeper on any of these" — it signals confidence and invites questions rather than forcing content on the audience.
Handling Q&A — the three answers to memorize
Prepare 2-sentence answers for: (1) "Why not a Transformer?" — cost is O(L²) and locality inductive bias is lost; CNN is O(L) and architecturally matched. (2) "Why not U-Net?" — pooling loses positional precision; we need to know the exact degree bin. (3) "Why not just use more L1 data?" — sparsity collapse is a mathematical property of the loss landscape, not a data problem. Having these locked in lets you answer instantly and pivot to the next question.
If asked something you don't know
"That's a good direction I haven't benchmarked yet — the next experiment I'd run is X" is stronger than guessing. For BeamNet: if asked about real-time latency, you can say "inference is a single forward pass on a 180-element vector — on GPU it's microseconds; I haven't profiled exact numbers yet." Honesty + next step = credibility.
